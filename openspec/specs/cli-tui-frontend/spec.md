# cli-tui-frontend 规范

## 目的
定义全屏终端前端、首次使用安装向导、TUI 内 slash commands、生成进度展示和依赖检查要求，使零基础用户可以不记忆传统 CLI 参数也能完成科普文章生成。

## 需求

### 需求: 全屏 TUI 入口
系统必须提供 `pop-agent tui` 命令，启动全屏终端界面，并允许用户不记忆传统 CLI 参数也能完成科普文章生成。

#### 场景: 零基础用户生成文章
- **当** 用户启动 `pop-agent tui`
- **并且** 在生成页填写主题、目标读者和用户 ID
- **并且** 点击生成按钮
- **那么** 系统必须运行现有多 Agent 生成流程
- **并且** 在界面中显示运行 ID、最终文章路径和文章内容

### 需求: 首次使用安装向导
系统必须在缺少用户配置且没有可用环境变量时提供安装向导，支持 DeepSeek、OpenAI-compatible 和 Mock 配置。

#### 场景: 保存 DeepSeek 配置
- **当** 用户在安装向导中选择 DeepSeek 并填写 API key
- **那么** 系统必须保存 openai-compatible 后端、DeepSeek base URL、模型名和 API key 到用户级配置文件
- **并且** 后续 `load_settings()` 必须能读取该配置

#### 场景: 使用 Mock 配置
- **当** 用户选择 Mock 模式
- **那么** 系统必须保存 mock 后端配置
- **并且** 生成流程必须在不访问网络的情况下可运行

#### 场景: 连接测试失败仍可保存配置
- **当** 安装向导的真实模型连接测试因为 401、TLS EOF 或超时失败
- **那么** 系统必须显示针对认证或网络的可读诊断
- **并且** 用户必须能够选择跳过测试并保存当前配置

### 需求: TUI 内 slash commands
系统必须在 TUI 底部提供 slash command 输入框，并支持基础命令。

#### 场景: 执行帮助命令
- **当** 用户输入 `/help`
- **那么** 系统必须显示可用命令、别名和参数提示

#### 场景: 执行生成命令
- **当** 用户输入 `/generate --topic "黑洞" --audience "初中生" --backend mock`
- **那么** 系统必须运行生成流程
- **并且** 输出运行 ID、文章路径和最终文章内容

#### 场景: 执行安装命令
- **当** 用户输入 `/install`
- **那么** TUI 必须打开安装向导

### 需求: TUI 进度展示
系统必须在 TUI 生成过程中展示多 Agent 阶段进度。

#### 场景: 查看生成阶段
- **当** 生成流程正在运行
- **那么** TUI 必须依次显示读取记忆、教师草稿、学生反馈、反馈聚合、事实检查、编辑和记忆更新等阶段信息

### 需求: TUI 安装向导依赖检查
系统必须在 TUI 安装向导中展示运行依赖状态，并允许用户触发依赖安装或修复。

#### 场景: 检查依赖状态
- **当** 用户打开安装向导
- **那么** 系统必须显示运行依赖是否已安装

#### 场景: 安装修复依赖
- **当** 用户点击安装/修复依赖
- **那么** 系统必须执行项目依赖安装流程
- **并且** 在安装完成后刷新依赖状态或显示失败信息
