## 需求
### 需求:多 Agent 文字科普生成
系统必须使用教师、学生、反馈聚合、事实检查和编辑 Agent 组成的流水线生成文字科普文章。

#### 场景:生成完整文章
- **当** 用户通过 CLI 或 API 提供主题、目标读者和用户 ID
- **那么** 系统必须生成最终科普文章、迭代摘要、评分和 artifact 列表

### 需求:迭代评估与修订
系统必须让学生 Agent 基于目标读者和用户记忆评估教师稿，并让教师 Agent 根据高影响反馈修订。

#### 场景:存在高影响困惑
- **当** 学生反馈中存在 impact 大于等于 7 的困惑
- **那么** 系统必须进入下一轮教师修订，直到达到停止条件或最大轮数

### 需求:停止条件
系统必须在理解评分达标、无高影响困惑、事实检查无阻断问题或达到最大轮数时停止迭代。

#### 场景:达到最大轮数
- **当** 迭代次数达到配置的最大轮数
- **那么** 系统必须停止修订并在报告中保留未解决问题

### 需求:运行记录落盘
系统必须为每次生成创建独立运行目录，保存输入、每轮草稿、反馈、最终文章和状态摘要。

#### 场景:生成运行目录
- **当** 一次生成任务启动
- **那么** 系统必须创建包含 state.json、drafts、feedback、final 和 report 的运行记录
