# 添加全屏 CLI TUI 前端

## 背景
当前系统已有 Typer 命令行、FastAPI 预留接口、多 Agent 生成服务和 Markdown 记忆系统，但零基础用户仍需要理解多条命令、环境变量和数据目录。为了让用户能直接进入“选择主题 -> 生成 -> 查看记忆/运行记录”的完整流程，需要一个全屏 TUI 前端，并提供首次使用安装向导。

## 目标
- 提供 `pop-agent tui` 全屏终端界面，直接复用现有生成服务、记忆系统和运行记录。
- 提供首次使用安装向导，支持 DeepSeek、OpenAI-compatible 和 Mock 配置，并将用户级配置写入安全配置目录。
- 支持基础 slash commands，使高级用户能在 TUI 内快速执行生成、记忆、运行记录、设置和帮助命令。
- 保持 CLI/TUI/API 共用同一核心服务层，为未来 Web 前端保留清晰接口。

## 非目标
- 本变更不实现 Web 前端。
- 本变更不实现图片/视频生成，只保留文字生成和现有 artifact 扩展协议。
- 本变更不引入常驻后台 daemon；“install”在本阶段实现为可重复运行的安装/配置向导。

## 借鉴点
参考 Claude Code 泄露仓库中可借鉴的模式：命令注册表、slash command 解析和本地安装配置流程。实现时采用相同思想，但使用本项目自己的 Python/Typer/Textual 架构。
