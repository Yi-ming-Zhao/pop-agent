## ADDED Requirements

### 需求:Markdown 用户认知记忆
系统必须以 Markdown 文件保存用户与科普相关的长期记忆，包括认知能力、知识掌握、误解和表达偏好。

#### 场景:读取用户记忆
- **当** 生成任务带有 user_id
- **那么** 系统必须读取该用户的 Markdown 记忆并用于学生 Agent 初始化

### 需求:自动记忆更新
系统必须在每次生成结束后自动提取并写入新的用户记忆，同时保留来源运行和变更记录。

#### 场景:自动写入新知识掌握情况
- **当** 学生反馈或用户输入显示用户掌握或误解某个知识点
- **那么** 系统必须将该信息写入对应 Markdown 记忆并记录 changelog

### 需求:记忆搜索索引
系统必须为 Markdown 记忆维护 JSON 搜索索引，支持按主题、标签、关键词和置信度检索。

#### 场景:初始化学生 Agent
- **当** 学生 Agent 评估某个主题稿件
- **那么** 系统必须搜索相关记忆并提供给学生 Agent，而不是加载全部记忆全文

### 需求:记忆可查看
系统必须提供 CLI 和 API 能力查看、搜索和更新用户记忆。

#### 场景:命令行搜索记忆
- **当** 用户运行 memory search 命令
- **那么** 系统必须返回匹配记忆的文件、标题、摘要、标签和置信度

### 需求:真实运行后的认知记忆沉淀
系统必须在真实模型生成流程结束后，将学生 Agent 暴露出的理解水平和高影响困惑沉淀为后续可检索的用户记忆。

#### 场景:生成结束写入困惑
- **当** 学生 Agent 在最终一轮反馈中返回 impact 大于等于 7 的困惑
- **那么** 系统必须写入 misconceptions 记忆，并保留 topic、source_run 和 confidence
